BUILD PROCESS OVERVIEW
════════════════════════════════════════════════════════════════════════════════

Weekly Automated Build (Every Sunday 6:00 AM UTC):

  1. GitHub Actions triggers container-build.yml
  2. Build Docker image with Alpine Linux 3.19 base
  3. Include all 9 CLI tools (AWS, Azure, Terraform, Packer, Ansible, yq, jq, etc)
  4. Push to Docker Hub with three tags:
     • Semantic: v0.3.0
     • Date-based: 20260504
     • Latest: latest
  5. Generate SBOM and provenance data
  
  → On Success: Automatically trigger update-docs.yml
  
  6. Extract versions from running container
  7. Generate/update README.md with current versions
  8. Commit with [skip ci] flag (prevents new build)
  9. Push back to repository


VERSIONING EXPLAINED
════════════════════════════════════════════════════════════════════════════════

Three-tier tagging system provides maximum flexibility:

  • v0.3.0 (Semantic) - Update VERSION file before major releases
  • 20260504 (Date) - Automatically generated, helpful for tracking
  • latest - Always points to most recent successful build

Example:
  docker pull techlotse/codestream-cloudcli:v0.3.0
  docker pull techlotse/codestream-cloudcli:20260504
  docker pull techlotse/codestream-cloudcli:latest


NO BUILD LOOPS PROTECTION
════════════════════════════════════════════════════════════════════════════════

The automation is designed to prevent circular build triggers:

  • Container build → triggers → documentation update
  • Documentation update includes [skip ci] flag
  • [skip ci] prevents triggering new container build
  
Result: Clean separation, no endless loops

TROUBLESHOOTING
════════════════════════════════════════════════════════════════════════════════

If first build fails:
  1. Check GitHub Actions logs for error details
  2. Verify Docker Hub credentials are set
  3. Check file permissions in repository
  4. Retry via workflow_dispatch

If README doesn't update:
  1. Check update-docs.yml logs for version extraction errors
  2. Verify image is available on Docker Hub
  3. Check git push permissions

If build loop occurs:
  1. Verify [skip ci] flag is in commit message
  2. Check workflow_run trigger configuration
  3. Ensure update-docs.yml is condition-protected
